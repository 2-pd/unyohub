_/_/_/_/ 路線系統情報入力規則 _/_/_/_/

※「railroad_info_サンプル.json」とあわせて確認してください。なお、サンプルには南海本線系統の路線系統情報を使用しています。

(1) ファイル形式
路線系統情報ファイルは「JSON」の規格に沿って記述し、ファイル名は「railroad_info.json」とします。
JSONファイルは一般的なテキストエディタで編集可能ですが、編集時は文字コードが「UTF-8(BOMなし)」に設定されていることを確認してください。

なお、路線系統情報ファイルでは、英数字や丸括弧を全て半角で入力する必要があります。


(2) 全体の構造
路線系統情報ファイルは、1つの路線系統についての情報を記載した単一の連想配列(以下、「基底の連想配列」と呼びます)からなり、基底の連想配列は以下のキーとそれに対応する値を有する必要があります。
基底の連想配列のキーのうち、省略可能であることが明記されていないものはアプリの正常な動作に必要不可欠であるため、入力漏れがないよう特に注意してください。

{
    "railroad_name" : 路線系統表示名,
    "main_color" : 路線系統のテーマカラー,
    "railroad_icon" : 路線系統のアイコンデータ,
    "related_railroads" : 関連する路線系統の情報(キー「joined_railroads」を省略する場合は省略可能),
    "related_railroads_order" : 関連する路線系統の路線系統識別名(キー「related_railroads」を省略する場合は省略),
    "joined_railroads" : 直通する路線系統の路線系統識別名(他の路線系統に属する運用が乗り入れてこない場合は省略可能),
    "lines" : 路線情報,
    "lines_order" : この路線系統に属する路線の路線識別名,
    "joined_lines_order" : 乗り入れ先路線系統に属する路線の路線識別名(他の路線系統に乗り入れていない場合は省略可能),
    "train_color_rules" : 列車表示色の判定規則,
    "deadhead_train_number" : 回送列車の列車番号にマッチする正規表現,
    "alias_of_forward_direction" : 前位側(奇数側)の呼称,
    "forward_direction_is_east" : 編成表左端の先頭車が東向きか(東向きでない場合は省略可能),
    "description" : 路線系統データの説明文(省略可能),
    "editors" : データ製作者の情報(省略可能),
    "license_text" : ライセンス文(省略可能),
    "acknowledgment" : 謝辞の文(省略可能),
    "related_links" : 関連リンク(省略可能)
}


(3) 路線系統の基本情報
基底の連想配列のキー「railroad_name」「main_color」「railroad_icon」の値は路線系統一覧画面などで表示される路線系統の基本情報となります。
それぞれ、以下のように値を指定してください。

"railroad_name" ・・・ 路線系統の表示名を文字列として指定します。(例: "南海本線系統")
"main_color" ・・・ 路線系統のテーマカラーの色コードをハッシュ記号に続けて16進数6桁で文字列として指定します。(例: "#3399ff")
"railroad_icon" ・・・ 路線系統のアイコン画像をBase64方式で文字列に変換して埋め込みます。変換にはサードパーティ製のオンラインツール等を使用してください。アイコン画像の形式はファイルサイズ抑制の観点からWebP形式とすることが推奨されています。


(4) 関連する路線系統の情報
同一鉄道事業者の別路線系統や相互乗り入れを行っている路線系統など、自路線系統と関連のある路線系統が存在する場合は、基底の連想配列のキー「related_railroads」と「related_railroads_order」を設定することにより編成表画面などで関連を明示することができます。
この設定で関連を明示していない路線系統は後述するキー「joined_railroads」の値に設定することができないため、他路線系統と直通する列車の運転されている路線系統において関連する路線系統の明示設定は必須となります。

「related_railroads」の値は連想配列であり、この連想配列は路線系統識別名(例: "koya")をキー、その路線系統識別名に対応する路線系統の情報を格納した連想配列を値として持ちます。
各路線系統の情報を格納する連想配列は以下のキーを持ちます。

{
    "railroad_name" : 表示名,
    "main_color" : テーマカラー,
    "show_link_in_formation_table" : 編成表で「関連項目」欄に表示するか否か(表示しない場合は省略可能)
}

それぞれのキーに対応する値は以下のように設定してください。

"railroad_name" ・・・ 路線系統の表示名を文字列として指定します。(例: "高野線・泉北線系統")
"main_color" ・・・ 路線系統のテーマカラーの色コードをハッシュ記号に続けて16進数6桁で文字列として指定します。(例: "#33cc66")
"show_link_in_formation_table" ・・・ この路線系統を編成表の「関連項目」欄に表示する場合は真偽値の「true」を指定します。表示する必要がない場合はこの項目を省略するか、真偽値の「false」を指定してください。

さらに、「related_railroads」に記載した路線系統は、その路線系統識別名を本アプリケーション上で表示させたい順に配列で列挙して、基底の連想配列のキー「related_railroads_order」の値に設定する必要があります。(例: ["koya", "choshi"])
JSONの仕様では、連想配列の要素の順序がブラウザの内部処理の過程で崩れてしまう可能性を考慮しなければならないことになっているため、路線系統情報ファイルにはこのように順序を記述するための項目が複数存在しています。


(5) 他の路線系統との直通設定
本アプリケーションでは各種データが路線系統ごとに独立して管理されるため、他の路線系統と直通する列車の運転されている路線系統では、直通先路線系統とデータを連携させるための設定(以下、「直通設定」と呼びます)が必要となります。

直通設定では、まず、基底の連想配列にキー「joined_railroads」を設け、その値の配列で直通先路線系統の路線系統識別名を列挙します。(例: ["hanwa", "yamatoji"])
この際、直通先路線系統が1つしかなければ、要素が1つのみの配列を記載します。

直通設定を行う場合、この他に乗り入れ先路線系統に属する路線の情報も記載する必要がありますが、これについては後述する「路線情報」の項目にて説明します。


(6) 路線情報
基底の連想配列のキー「lines」の値には、自路線系統に属する全路線、ならびに、それらの路線内の駅を発着する列車が直通する直通先路線系統の路線についての情報を記述します。
直通先路線系統の路線は、自路線系統の路線を発着する列車が乗り入れるもののみ記載します。(例えば、JR西日本の環状線系統では、阪和線系統の阪和線や関西空港線から列車が乗り入れてくるため阪和線と関西空港線を記載する必要がありますが、大阪環状線に直通する列車がなく、環状線系統に所属する編成による定期運用も存在していない羽衣線は記載する必要がありません。)
「lines」の値は連想配列であり、この連想配列は路線識別名(例: "honsen")をキー、その路線識別名に対応する路線の情報を格納した連想配列を値として持ちます。

各路線の情報を格納する連想配列は、自路線系統の路線・直通先路線系統の路線共に以下の構造となります。
キー「line_name」「line_color」「inbound_forward_direction」は、その路線が自路線系統の路線であるか否かを問わず、省略することができません。

{
    "line_name" : 路線名,
    "line_color" : 路線のテーマカラー,
    "line_symbol" : 路線記号または路線の略称(省略可能),
    "affiliated_railroad_id" : 路線が属する路線系統の路線系統識別名(直通先路線系統に属する路線でない場合は省略可能),
    "inbound_forward_direction" : 上り列車の進行方向と編成の前位側(奇数向き)方向が一致するかどうか,
    "clockwise_is_inbound" : 上り列車が時計回り運転となるか(環状運転していない路線では省略可能),
    "double_station_spacing" : 走行位置モードで駅間を2行で表示させるか(1行で表示させる場合や直通先路線系統の路線の場合は省略可能),
    "stations" : 駅情報
}

それぞれのキーに対応する値は以下のように設定してください。

"line_name" ・・・ 路線名を文字列として指定します。(例: "南海本線")
"line_color" ・・・ 路線のテーマカラーの色コードをハッシュ記号に続けて16進数6桁で文字列として指定します。(例: "#3399ff")
"line_symbol" ・・・ 路線記号等を2文字以内(英数字以外の文字では1文字のみ)の文字列で指定します。この項目を省略した場合、本アプリケーションは路線名の1文字目を路線記号の代わりとして表示します。(例: "本")
"affiliated_railroad_id" ・・・ 直通先路線系統に属する路線の場合は真偽値の「true」を指定します。自路線系統に属する路線では省略するか真偽値の「false」を指定してください。
"inbound_forward_direction" ・・・ この路線で上り列車の進行方向と編成の前位側方向が一致する(つまり、上り列車で編成表の左端の車両が先頭に立つ)場合は真偽値の「true」を、一致しない(つまり、上り列車で編成表の右端の車両が先頭に立つ)場合は真偽値の「false」を指定します。
"clockwise_is_inbound" ・・・ 環状運転を実施している路線の場合、上り列車が時計回りの運転であれば真偽値の「true」を、下り列車が時計回りの運転であれば真偽値の「false」を指定します。環状運転を実施していない路線の場合は「null」を指定してください。省略時は「null」が指定されたものとみなされます。
"double_station_spacing" ・・・ 各駅間を1行で表示している場合、多くの駅間に2列車以上が存在するような高密度運転を実施している路線では、複数の列車が走行位置画面で同一位置に表示されるため利便性が低下します。これを防ぐには、この値に真偽値の「true」を指定して各駅間を2行で表示させるようにしてください。省略時は真偽値の「false」が指定されたものとみなされ、各駅間は1行で表示されます。
"stations" ・・・ 各駅の情報を起点側から順に記載してください。詳細は後述する「駅情報」の項目にて説明します。


(7) 駅情報
各路線情報のキー「stations」の値には、その路線に属する駅の情報を起点側から終点に向かう順(下り列車の停車・通過順)で記述します。
「stations」の値は配列であり、この配列の各要素が1つの駅(または信号場)に対応します。

各要素は以下のキーを持つ連想配列です。

{
    "station_name" : 駅名,
    "station_name_kana" : 駅名のふりがな(直通先路線系統に属する路線の路線情報中では省略可能),
    "station_initial" : 駅名の1文字表記(単一の路線に属し、始発・終着とする列車がない駅では省略可能),
    "is_signal_station" : 信号場か否か(信号場でなければ省略可能),
    "is_major_station" : 主要駅か否か(主要駅でなければ省略可能),
    "connecting_lines" : 分岐路線情報(分岐駅でないか、直通先路線系統に属する路線の路線情報中では省略可能),
    "connecting_railroads" : 乗り換え可能な路線系統の情報(直通先路線系統に属する路線の路線情報中では省略可能)
}

それぞれのキーに対応する値は以下のように設定してください。

"station_name" ・・・ 駅名を文字列として指定します。副駅名を含める必要はありません。(例: "和歌山市")
"station_name_kana" ・・・ 駅名のふりがなを平仮名で指定します。漢字を含まない駅名の場合でもこの項目を省略することはできません。(例: "わかやまし")
"station_initial" ・・・ 駅名の1文字表記を文字列として指定します。通常は駅名の頭文字を使用しますが、頭文字が他の駅と被る場合には別の文字を使用して重複を回避してください。複数の路線に属する駅でなく、かつ、当該の駅を始発または終着とする列車が存在しない場合には省略可能です。(例: "市")
"is_signal_station" ・・・ 駅ではなく信号場である場合は真偽値の「true」を指定してください。駅である場合は、この項目を省略するか、真偽値の「false」を指定します。
"is_major_station" ・・・ 主要駅である場合は真偽値の「true」を指定してください。主要駅でない場合は、この項目を省略するか、真偽値の「false」を指定します。
"connecting_lines" ・・・ 当該の駅が、自路線系統の別路線や直通先路線系統の路線にも所属している場合、それらの路線の情報を記載します。詳細は後述する「分岐路線情報」の項目を参照してください。1つの路線のみに所属する駅ではこの項目を省略可能です。
"connecting_railroads" ・・・ 当該の駅から(直通列車のない)他の路線系統に乗り換えできる場合、それらの路線系統の情報を記載します。詳細は後述する「乗り換え可能な路線系統の情報」の項目を参照してください。本アプリケーションに登録されている他の路線系統への乗り換えが案内されていない駅ではこの項目を省略可能です。

◆ 分岐路線情報
分岐路線情報は、3つ以上の路線に属する駅に対応するため配列で記載する仕様となっており、この配列はそれぞれ1つの路線に対応する以下の連想配列を要素として持ちます。

{
    "line_id" : 路線識別名,
    "directions" : 分岐方向
}

この連想配列の各キーに対応する値は以下のように設定してください。

"line_id" ・・・ 分岐または交差している路線の路線識別名を文字列として指定します。(例: "wakayamakosen")
"directions" ・・・ 分岐または交差している路線の分岐・交差方向を、「TL」(左上)、「T」(上)、「TR」(右上)、「L」(左)、「R」(右)、「BL」(左下)、「B」(下)、「BR」(右下)から1つ以上選んで配列として設定します。(例: ["B"])

◆ 乗り換え可能な路線系統の情報
分岐路線情報は、2つ以上の路線系統に乗り換え可能な駅に対応するため配列で記載する仕様となっており、この配列はそれぞれ1つの路線系統に対応する以下の連想配列を要素として持ちます。

{
    "railroad_id" : 路線系統識別名,
    "lines" : 乗り換え可能な路線の情報(省略可能)
}

この連想配列の各キーに対応する値は以下のように設定してください。

"railroad_id" ・・・ 乗り換え可能な路線系統の路線系統識別名を文字列として指定します。(例: "wakayama")
"lines" ・・・ 当該路線系統の連絡先駅を明示する場合は、以下の連想配列(それぞれ1路線に対応)を要素として持つ配列を指定します。

{
    "line_id" : 路線識別名,
    "line_name" : 路線名,
    "station_name" : 駅名(自路線系統の駅名と同一である場合は省略可能)
}

この連想配列のキーのうち、「line_name」の値については、連絡先路線系統の路線系統情報ファイルに記載されている当該路線の路線名と完全に一致する必要はありません。自路線系統の車内アナウンスなどで連絡先路線を略称で案内している場合はその略称を使用してもよいでしょう。


(8) 路線表示順序の設定
基底の連想配列のキー「lines_order」と「joined_lines_order」の値には、それぞれ自路線系統の路線と直通先路線系統の路線の路線識別名を本アプリケーション内の路線選択画面に表示させたい順序で列挙した配列を指定します。(例: ["honsen", "kukosen", "shiomibashisen", "takashinohamasen", "tanagawasen", "kadasen", "wakayamakosen"])
直通先路線系統の路線は、路線情報と同様、自路線系統の路線を発着する列車が乗り入れるもののみ記載します。

直通設定を行わない場合、「joined_lines_order」は省略してください。


(9) 列車表示色の判定規則
基底の連想配列のキー「train_color_rules」の値には、列車表示色の判定規則を指定します。
列車表示色の判定規則は、それぞれ1つの列車表示色に対応する連想配列が要素として含まれた配列であり、要素となる連想配列は以下のキーを持ちます。
列車の色分けが必要がない場合でもキー「train_color_rules」を省略することはできないため、そのようなときは値の配列を空としてください。

{
    "pattern" : 正規表現,
    "color" : 列車表示色,
    "subject" : 判定対象(省略可能)
}

それぞれのキーに対応する値は以下のように設定してください。

"pattern" ・・・ 当該の表示色で表示させたい列車の列車番号または列車種別にマッチする正規表現を文字列として指定します。(例: "^3")
"color" ・・・ 列車表示色の色コードをハッシュ記号に続けて16進数6桁で文字列として指定します。(例: "#008000")
"subject" ・・・ 正規表現の判定対象を、列車番号とする場合は文字列"train_number"を、列車種別とする場合は"train_type"を指定します。省略時は"train_number"が指定されたものとみなされます。

なお、ある列車の列車番号や列車種別が判定規則の配列中に存在する複数色分の正規表現にマッチする場合は、それらのうち配列中の順序で最も前にある色が当該列車の表示色として適用されます。


(10) 回送列車の列車番号にマッチする正規表現
基底の連想配列のキー「deadhead_train_number」の値には、回送列車の列車番号にマッチする正規表現を文字列として設定します(例: "^[89回]")
ここで設定された正規表現にマッチする列車は、種別が指定されていない場合(時刻表に記載のない列車が運用表に入力されたものなど)でも走行位置画面などで回送列車として正しく表示されます。
なお、回送列車の存在しない路線系統であってもキー「deadhead_train_number」を省略することはできないため、値として営業列車の列車番号にマッチしない適当な正規表現("^回送"など)を指定しておく必要があります。


(11) 編成表メタ情報
基底の連想配列のキー「alias_of_forward_direction」と「forward_direction_is_east」の値は、編成表を表示する際などに必要となるメタ情報です。
それぞれ、以下のように値を指定してください。

"alias_of_forward_direction" ・・・ 編成表左側向きの呼称を文字列で指定します。その方向に向かって走行する際の終着駅の駅名を使った表現が一般的です。(例: "難波方")
"forward_direction_is_east" ・・・ 編成表左端の先頭車が東向きか否かを真偽値で入力します。編成表の左→右の向きと路線の東→西の向きが概ね一致している場合のみ真偽値の「true」を指定してください。省略時は真偽値の「false」が指定されたものとみなされます。路線が南北に伸びている場合や環状路線である場合など、路線の配置がそもそも東西方向と無関係なときも「false」を指定することが適正です。


(12) 路線系統データの説明文
基底の連想配列のキー「description」の値には、路線系統データの説明文を文字列として入力します。
この説明文には、路線系統の簡単な紹介に加え、利用者にとって注意すべき点なども記載するとよいでしょう。データ製作者の情報とデータのライセンス、協力者への謝辞、関連リンクは専用の入力項目があるため、ここに含める必要はありません。
説明文の文字列中に改行を配置したい場合は、当該箇所へ改行コードのエスケープシーケンス「\n」を入力してください。


(13) データ製作者の情報
基底の連想配列のキー「editors」の値には、編成表や運用表、時刻表などのデータ製作に関わったメンバーの情報を入力します。
この値は、それぞれ1人のデータ製作者に対応する連想配列を要素として持つ配列であり、要素となる連想配列は以下のキーを持ちます。

{
    "editor_name" : ハンドルネーム,
    "editor_url" : webサイト等のURL(省略可能),
    "introduction_text" : 紹介文(省略可能)
}

それぞれのキーに対応する値は以下のように設定してください。

"editor_name" ・・・ データ製作者のハンドルネームを文字列で指定します。(例: "名無しの権兵衛")
"editor_url" ・・・ データ製作者のwebサイト等(ブログなど)のURLを文字列で指定します。データ製作者が自身のwebサイトを持っていない場合には省略してください。(例: "https://www.example.jp/")
"introduction_text" ・・・ データ製作者についての簡単な紹介や、製作したデータの範囲などを説明した文を文字列として指定してください。


(14) ライセンスと謝辞
基底の連想配列のキー「license_text」と「acknowledgment」の値は、それぞれ、路線系統データのライセンスと協力者への謝辞の文字列としてください。
利用条件として出典の表記が要求されているオープンデータをもとに時刻表データなどを作成した場合は、このライセンス文中でオープンデータの出典を明示する必要があります。
なお、これらの文字列中でも、路線系統データの説明文と同様、改行コードのエスケープシーケンス「\n」を入力することで改行を配置することが可能です。


(15) 関連リンク
基底の連想配列のキー「related_links」の値には、路線系統に関連したwebページへのリンクを入力します。
この値は、それぞれ1件のページに対応する連想配列を要素として持つ配列であり、要素となる連想配列は以下のキーを持ちます。
本アプリケーションで提供している運用表や時刻表等のデータをPDFなど他の形式でも配布している場合は、その配布ページのリンクを掲載するとよいでしょう。

{
    "link_url" : リンクURL,
    "link_text" : リンクテキスト,
    "link_description" : リンクの説明(省略可能)
}

それぞれのキーに対応する値は以下のように設定してください。

"link_url" ・・・ リンクのURLを文字列として指定します。(例: "https://www.example.jp/nankai_honsen.pdf")
"link_text" ・・・ リンクとして表示されるテキストを文字列として指定します。(例: "運用表PDFファイル")
"link_description" ・・・ リンク先についての短い紹介文を文字列として指定します。(例: "印刷用の運用表データです")
